# Quality Gates (Codex)

Run this checklist before advancing phases.

## Gate A — Factual integrity
- [ ] Facts are clearly separated from assumptions.
- [ ] Claims are traceable to code/interview evidence.
- [ ] Unknowns are explicitly listed in an Open Questions section.
- [ ] 7 R's classification is justified with evidence (Phase 1+).

## Gate B — Business alignment
- [ ] Domain expert feedback is reflected.
- [ ] Recommendations map to stated goals/constraints.
- [ ] Risks include business and operational impact.
- [ ] Application landscape rationalization was performed before architecture design (Phase 3).
- [ ] Every major recommendation is tagged [Tactical] or [Strategic].

## Gate C — Engineering viability
- [ ] Architecture choices include trade-offs and red team critique (counter-arguments, risks, mitigations).
- [ ] Migration strategy includes rollback/safety approach.
- [ ] Testing expectations are explicit (unit, integration, API, end-to-end for critical flows).
- [ ] Modern capabilities opportunities trace back to specific pain points (not tool-driven).
- [ ] Timeline estimates account for organization size (logistics multiplier applied).
- [ ] SEO/AI discoverability assessed for public-facing applications.

## Gate D — Governance/compliance
- [ ] Relevant regulatory framework assumptions are stated.
- [ ] Security risks and remediation priorities are present.
- [ ] Cross-document terminology and scope are consistent.
- [ ] Consolidated cross-application assessment completed (multi-app engagements).
